# 参考资料

本文档记录仓库内各 Agnes 接入 skill 的资料来源，方便后续更新时按来源逐项复核。

## 通用 Agnes API

- [Agnes AI 中文概述](https://agnes-ai.com/zh-Hans/docs)
  - 用途：确认 Agnes API 的 OpenAI 兼容接入方式、Base URL、认证方式和多模态能力范围。
- [Agnes 2.0 Flash 文档](https://wiki.agnes-ai.com/zh-Hans/docs/agnes-20-flash)
  - 用途：确认 `agnes-2.0-flash` 的 chat completions、图像 URL 输入、工具调用、Thinking、上下文和输出规格。
- [Agnes Image 2.1 Flash 文档](https://wiki.agnes-ai.com/zh-Hans/docs/agnes-image-21-flash)
  - 用途：确认 `agnes-image-2.1-flash` 的 `/v1/images/generations` 端点、文生图、图生图、`extra_body.image`、`response_format`、Base64 输出和图像尺寸注意事项。
- [Agnes-help-skill 原参考项目](https://github.com/lj1270998580-crypto/Agnes-help-skill/blob/main/SKILL.md)
  - 用途：对照 Agnes 社区 skill 中已整理的接入经验和常见错误，保留关键图像参数坑位，删除冗余的视频、平台矩阵和版本提醒内容。

## QwenClaw / QwenPaw Skill

- [QwenPaw Skills 文档](https://qwenpaw.agentscope.io/docs/skills)
  - 用途：确认 QwenPaw/QwenClaw skill 目录结构、`SKILL.md` front matter 要求、技能池、工作区技能副本、GitHub URL 导入和 `metadata.requires.env` 约定。
- [QwenPaw Models 文档](https://qwenpaw.agentscope.io/docs/models.zh.md)
  - 用途：后续更新模型配置说明时复核 Settings -> Models、自定义 provider、默认模型和模型切换逻辑。

对应文件：

```text
agnes-qwenclaw/SKILL.md
```

```text

```

  - 用途：确认 `models.providers`、provider id、`baseUrl`、`apiKey`、`api: "openai-completions"`、模型注册和默认模型引用方式。


## 后续更新建议

1. 先复核 Agnes 官方文档，确认 Base URL、模型名、端点和参数位置是否变化。
2. 再复核各宿主工具官方文档，确认配置字段和 CLI 命令是否变化。
3. 最后对照社区 skill，只吸收实际接入经验和常见错误，不复制版本提醒、视频接口、平台矩阵等非本仓库目标内容。
4. 更新 skill 后运行 `git diff --check`，并检查是否误写真实 API Key。
